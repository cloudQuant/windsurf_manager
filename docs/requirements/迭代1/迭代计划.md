# Windsurf Manager 迭代1 — 简易迭代计划

## 目标

实现一个前后端分离的 Windsurf 账号管理工具，支持：

1. **账号管理** — 增删改查 Windsurf 账号信息
2. **账号切换** — 一键切换账号，同时实现：
   - **本地 Windsurf IDE 登录切换**（替换 `state.vscdb` 中的认证数据）
   - **官网 Web 登录**（通过 Playwright 自动化浏览器登录 windsurf.com）
3. **额度查询** — 查看各账号的剩余额度/用量

## Windsurf 认证机制分析

通过逆向分析，Windsurf 认证数据存储位置如下：

| 存储位置 | 路径 / 键 | 内容 |
|---|---|---|
| state.vscdb | `windsurfAuthStatus` | API Key (`sk-ws-...`) + 模型配置 JSON |
| state.vscdb | `windsurf_auth-{用户名}` | 当前登录用户标识 |
| state.vscdb | `windsurf_auth-{用户名}-usages` | 扩展使用记录 |
| state.vscdb | `secret://...windsurf_auth.sessions` | 加密的会话数据 |
| state.vscdb | `secret://...windsurf_auth.apiServerUrl` | 加密的 API Server URL |
| state.vscdb | `codeium.windsurf-windsurf_auth` | 认证标记 |
| macOS Keychain | "Windsurf Safe Storage" | 用于加密/解密 state.vscdb 中 secret 字段的密钥 |

**关键路径**: `~/Library/Application Support/Windsurf/User/globalStorage/state.vscdb`

## 技术选型

| 层 | 技术 | 说明 |
|---|---|---|
| 前端 | Vue 3 + Vite + Element Plus | 轻量 SPA，快速搭建 |
| 后端 | Python + FastAPI | 异步高性能，自带 OpenAPI 文档 |
| 数据存储 | SQLite（通过 SQLAlchemy） | 零部署成本，单文件数据库 |
| 浏览器自动化 | Playwright (Python) | 官网登录 + 额度抓取 |
| 本地切换 | sqlite3 直接操作 state.vscdb | 替换认证条目 |
| 前后端通信 | REST JSON API | 简单直接 |

## 任务拆分

### Phase 1：后端基础 — 数据模型 & 接口（预计 1h）

1. **项目脚手架**
   - 初始化 `backend/` 目录，创建 `requirements.txt`
   - 依赖：`fastapi`, `uvicorn`, `sqlalchemy`, `pydantic`, `playwright`

2. **数据模型**
   - `Account` 表：
     - `id`, `name`(显示名), `email`, `password`(加密存储，用于官网登录)
     - `api_key`(Windsurf API Key `sk-ws-...`)
     - `auth_snapshot`(BLOB, state.vscdb 中所有认证相关键值的快照)
     - `is_active`(是否为当前账号)
     - `quota_total`, `quota_used`, `quota_updated_at`
     - `created_at`, `updated_at`

3. **API 接口**

   | 方法 | 路径 | 功能 |
   |---|---|---|
   | GET | `/api/accounts` | 获取所有账号列表 |
   | POST | `/api/accounts` | 新增账号 |
   | PUT | `/api/accounts/{id}` | 编辑账号 |
   | DELETE | `/api/accounts/{id}` | 删除账号 |
   | POST | `/api/accounts/{id}/activate` | 切换账号（本地 IDE + 官网） |
   | GET | `/api/accounts/{id}/quota` | 查询单个账号额度 |
   | POST | `/api/accounts/refresh-all-quotas` | 批量刷新所有账号额度 |
   | POST | `/api/accounts/import-current` | 从当前 Windsurf 导入活跃账号 |

### Phase 2：核心 — 账号切换引擎（预计 2h）

1. **导入当前账号**（`import-current`）
   - 读取 `state.vscdb` 中 `windsurfAuthStatus` 提取 `apiKey` 和用户名
   - 将所有 `windsurf_auth*` 相关键值序列化为快照保存到 `auth_snapshot`
   - 自动创建一条 Account 记录

2. **本地 IDE 切换**（`activate` 的第一步）
   - 关闭 Windsurf 进程（`pkill -f Windsurf` 或优雅信号）
   - 将目标账号的 `auth_snapshot` 写回 `state.vscdb`
   - 重启 Windsurf（`open -a Windsurf`）

3. **官网 Web 登录**（`activate` 的第二步）
   - 使用 Playwright 打开 `https://windsurf.com/account`（或登录页）
   - 填入目标账号的 email/password 完成登录
   - 登录成功后保持浏览器窗口打开，供用户查看

### Phase 3：额度查询（预计 1h）

1. **方案 A（优先）**：直接用已知的 `apiKey` 调用 Windsurf API 查询额度
   - `windsurfAuthStatus` 中已包含 API Key，可尝试调用 Windsurf 后端接口
2. **方案 B（兜底）**：用 Playwright 登录官网后抓取额度页面数据
   - 解析 `https://windsurf.com/account` 页面上的用量/额度信息

### Phase 4：前端（预计 1-2h）

1. **项目脚手架**
   - `npm create vue@latest` 初始化 `frontend/` 目录
   - 安装 Element Plus、Axios

2. **页面 & 组件**

   | 页面/组件 | 功能 |
   |---|---|
   | `AccountList` | 账号列表，显示名称、邮箱、是否活跃、额度、操作按钮 |
   | `AccountForm` | 新增/编辑账号弹窗（email + password + 可选 apiKey） |
   | `QuotaDisplay` | 额度进度条 / 数值展示 |
   | `ImportButton` | 一键从当前 Windsurf 导入账号 |
   | 顶部状态栏 | 当前活跃账号名称 + 剩余额度摘要 |

3. **交互流程**
   - 首次使用：点击「导入当前账号」从本地 Windsurf 自动导入
   - 添加新账号：填写 email + password，可选手动填入 apiKey
   - 切换账号：点击「切换」→ 后端自动完成 IDE 切换 + 官网登录
   - 查询额度：自动 / 手动刷新额度信息

### Phase 5：联调 & 收尾（预计 0.5h）

1. 前后端联调，处理 CORS
2. 添加错误提示（切换失败、登录失败等场景）
3. 编写 README，说明启动 & 使用方式
4. 基本冒烟测试

## 注意事项

- **密码安全**：账号密码在后端使用 Fernet 对称加密存储，密钥从环境变量读取
- **Windsurf 进程**：切换账号时需重启 Windsurf IDE 才能生效，会提前提醒用户
- **Playwright 浏览器**：首次使用需安装 Chromium（`playwright install chromium`）
- **state.vscdb 备份**：每次写入前自动备份，避免损坏用户数据
